คู่มือการใช้งาน

PMtools — เครื่องมือบำรุงรักษาเชิงป้องกัน Windows Server

เก็บสถานะเครื่องแม่ข่ายในรอบเดียว แล้วออกรายงานสองภาษาที่ส่งให้ผู้บริหารได้ทันที

เอกสารฉบับนี้อยู่ในโฟลเดอร์เดียวกับเครื่องมือ เปิดได้โดยไม่ต้องต่ออินเทอร์เน็ต และสั่งพิมพ์เป็น PDF ได้

สารบัญ

  1. เครื่องมือนี้ทำอะไร
  2. อ่านค่าอย่างเดียว ไม่แก้ไขระบบ
  3. เริ่มใช้งาน
  4. แบบไฟล์เดียว
  5. ส่งออกเป็น Word (.docx)
  6. เมนูเลือกสิ่งที่จะตรวจ
  7. วิธีอ่านรายงาน
  8. กราฟแนวโน้ม CPU และหน่วยความจำ
  9. หัวข้อที่ตรวจทั้งหมด
  10. ตรวจ ArcGIS Server
  11. การตั้งค่า
  12. สั่งงานจาก PowerShell
  13. ปัญหาที่พบบ่อย
  14. ข้อจำกัดที่ควรทราบ

1เครื่องมือนี้ทำอะไร

PMtools ตรวจสถานะเครื่องแม่ข่าย Windows ทุกหัวข้อที่ควรรู้ในการบำรุงรักษาเชิงป้องกัน แล้วสร้างไฟล์รายงาน HTML ภาษาไทยคู่อังกฤษ ที่จัดหน้าเรียบร้อยพอจะส่งให้ผู้บริหารหรือแนบหนังสือราชการได้โดยตรง

ใช้เวลาทั้งหมด ประมาณ 10 วินาที ตรวจ 14 หัวข้อ ตั้งแต่พื้นที่ดิสก์ หน่วยความจำ บริการที่หยุดทำงาน บันทึกเหตุการณ์ผิดพลาด ใบรับรองที่ใกล้หมดอายุ ไปจนถึงสถานะการสำรองข้อมูล

สิ่งที่ได้เมื่อรันเสร็จ

ไฟล์ใช้ทำอะไร
PM-Report.htmlรายงานที่ส่งให้ผู้ใช้ เปิดได้ทุกเครื่อง ไม่ต้องติดตั้งอะไร ไม่ต้องต่อเน็ต สั่งพิมพ์เป็น PDF ได้
PM-Data.jsonข้อมูลดิบทั้งหมดของรอบนั้น เก็บไว้เทียบย้อนหลังหรือส่งต่อระบบอื่น

ทั้งสองไฟล์อยู่ที่ Output\<ชื่อเครื่อง>_<วันเวลา>\ แต่ละรอบแยกโฟลเดอร์กัน ไม่ทับกัน (ถ้าใช้แบบไฟล์เดียว โฟลเดอร์รายงานจะอยู่ข้างตัวไฟล์โดยตรง ไม่มีชั้น Output\ คั่น)

ไม่ต้องติดตั้งอะไรบนเครื่องแม่ข่าย

เครื่องมือทำงานด้วย PowerShell 5.1 ที่ติดมากับ Windows อยู่แล้ว ไม่ต้องลง module เพิ่ม ไม่ต้องต่ออินเทอร์เน็ต คัดลอกโฟลเดอร์ไปวางแล้วรันได้เลย และลบทิ้งได้ทันทีเมื่อเลิกใช้

2อ่านค่าอย่างเดียว ไม่แก้ไขระบบ

เครื่องมือนี้ไม่เปลี่ยนค่าใด ๆ บนเครื่องแม่ข่าย

ทุกหัวข้อเป็นการอ่านสถานะเท่านั้น สิ่งเดียวที่เหลือไว้บนเครื่องคือโฟลเดอร์รายงาน

  • ไม่ตั้งค่า ไม่แก้ registry ไม่แตะ Group Policy
  • ไม่เริ่ม ไม่หยุด ไม่แก้ไขบริการใด ๆ
  • ไม่ติดตั้งอะไร ไม่ดาวน์โหลดอะไร
  • ไม่รีสตาร์ทเครื่อง และไม่สั่งอะไรที่ทำให้บริการสะดุด

เจ็ดหัวข้อที่ปิดไว้เป็นค่าเริ่มต้น

เป็นหัวข้อที่ต้องติดต่อออกนอกเครื่อง จึงปิดไว้ก่อนเพื่อความปลอดภัย

รหัสทำอะไรทำไมถึงปิดไว้เปิดเฉพาะครั้ง
WUถาม WSUS หรือ Microsoft Update ว่ามีรายการปรับปรุงอะไรค้างติดตั้ง ใช้เวลานานมาก และค้างได้บนเครือข่ายปิด ทดสอบแล้วค้างเกิน 10 นาที-Only WU
CONNทดสอบ ping ไปยังเกตเวย์และเซิร์ฟเวอร์ชื่อโดเมนทุกตัว ถ้าหน่วยงานมีอุปกรณ์ IPS/IDS อาจมองว่าเป็นการสแกนแล้วบล็อกเครื่อง-Only CONN
AGSตรวจ ArcGIS Server ผ่าน Administrator API — ดูหัวข้อที่ 10 ต้องเชื่อมต่อไปยัง ArcGIS Server และต้องตั้งค่าการเชื่อมต่อก่อนใช้งาน-Only AGS
AGSSVCตรวจบริการแผนที่บน ArcGIS Server — ดูหัวข้อที่ 10 เหตุผลเดียวกับ AGS-Only AGSSVC
AGSUSAGEอ่านรายงานการใช้งาน ArcGIS Server — ดูหัวข้อที่ 10 เหตุผลเดียวกับ AGS-Only AGSUSAGE
AGSDATAตรวจการเชื่อมต่อฐานข้อมูลที่ลงทะเบียนไว้กับ ArcGIS Server — ดูหัวข้อที่ 10 เหตุผลเดียวกับ AGS-Only AGSDATA
AGSLOGตรวจบันทึกเหตุการณ์ระดับ SEVERE/WARNING ของ ArcGIS Server — ดูหัวข้อที่ 10 เหตุผลเดียวกับ AGS-Only AGSLOG

อีก 14 หัวข้อที่เหลือเป็นการอ่านค่าในเครื่องล้วน ๆ ไม่ส่งแพ็กเก็ตออกนอกเครื่องเลยแม้แต่แพ็กเก็ตเดียว

3เริ่มใช้งาน

  1. คัดลอกโฟลเดอร์ไปวางบนเครื่องแม่ข่าย

    คัดลอกโฟลเดอร์ PMtools ทั้งโฟลเดอร์ ไม่ต้องแยกไฟล์ วางที่ไหนก็ได้ที่เขียนไฟล์ได้ เช่น D:\PMtools

    ถ้าได้มาเป็นไฟล์เดียวชื่อ PMtools-1.7.2.cmd ให้ข้ามไปหัวข้อที่ 4 แทน

  2. แก้ชื่อหน่วยงาน (ทำครั้งเดียว)

    เปิด Config\settings.json ด้วย Notepad แล้วแก้สามค่านี้ ชื่อที่ใส่จะไปขึ้นหัวรายงาน

    "Organization": {
  "NameTh": "กรมตัวอย่าง กระทรวงตัวอย่าง",
  "DepartmentTh": "ศูนย์เทคโนโลยีสารสนเทศ"
},
"Report": {
  "PreparedBy": "ชื่อผู้ตรวจสอบ"
}
  3. ดับเบิลคลิก Run-PM.cmd

    ระบบจะขอสิทธิ์ผู้ดูแลระบบ ให้กดอนุญาต จากนั้นจะขึ้นเมนูให้เลือกเวลาเก็บข้อมูล

  4. เลือกจากเมนู แล้วรอ

    ถ้าต้องการรายงานเร็ว ๆ กด Enter เฉย ๆ ได้เลย (= ตรวจทันที ไม่เก็บกราฟ) เมื่อเสร็จ รายงานจะเปิดขึ้นเองในเบราว์เซอร์

  5. บันทึกเป็น PDF เพื่อแนบหนังสือ

    ในเบราว์เซอร์กด Ctrl + P แล้วเลือกปลายทางเป็น Save as PDF สีและเส้นตารางจะติดไปครบ ไม่ต้องจัดหน้าเพิ่ม

ควรรันด้วยสิทธิ์ผู้ดูแลระบบ

ถ้ารันโดยไม่มีสิทธิ์ เครื่องมือจะไม่หยุดทำงาน แต่บางหัวข้อจะได้ข้อมูลไม่ครบ และจะมีแถบเตือนสีเหลืองแสดงไว้ในรายงานอย่างชัดเจน เพื่อไม่ให้ผู้อ่านเข้าใจผิดว่าข้อมูลสมบูรณ์แล้ว การใช้ Run-PM.cmd จะขอสิทธิ์ให้เองอยู่แล้ว

4แบบไฟล์เดียว

นอกจากแบบโฟลเดอร์ในหัวข้อที่แล้ว PMtools ยังแจกได้อีกแบบหนึ่งคือ ไฟล์เดียวจบ ขนาดราว 95 KB ที่รวมทุกอย่างไว้ข้างในหมดแล้ว เหมาะกับตอนส่งทางอีเมล หรือตอนที่เครื่องแม่ข่ายเข้าถึงได้จำกัดจนคัดลอกทั้งโฟลเดอร์ไม่สะดวก

ดูอย่างไรว่าได้แบบไหนมา

ถ้าได้มาเป็นให้ทำอย่างไร
โฟลเดอร์ PMtools ที่มีไฟล์หลายสิบไฟล์ ดับเบิลคลิก Run-PM.cmd ตามหัวข้อที่ 3
ไฟล์เดียวชื่อ PMtools-1.7.2.cmd ดับเบิลคลิกไฟล์นั้นได้เลย ไม่ต้องมีอะไรอยู่ข้าง ๆ

ทั้งสองแบบใช้งานเหมือนกันทุกอย่าง เมนูเดียวกัน หัวข้อที่ตรวจชุดเดียวกัน รายงานหน้าตาเดียวกัน ต่างกันแค่ที่วางรายงานเท่านั้น

วิธีใช้

  1. วางไฟล์ในโฟลเดอร์ที่เขียนไฟล์ได้

    เช่น D:\ตรวจเครื่อง\ หรือหน้า Desktop ก็ได้ ไม่ต้องเอาอะไรไปด้วย ไม่ต้องแตกไฟล์ก่อน

  2. ถ้าได้ไฟล์มาทางอีเมลหรือดาวน์โหลด ให้ปลดล็อกก่อน

    Windows จะติดเครื่องหมายไว้ว่าไฟล์มาจากเครื่องอื่น ให้คลิกขวาที่ไฟล์ → Properties → ติ๊ก Unblock ที่มุมล่าง → OK ถ้าคัดลอกมาทาง USB หรือ share ภายในหน่วยงาน มักไม่ต้องทำขั้นนี้

  3. ดับเบิลคลิก

    ระบบจะขอสิทธิ์ผู้ดูแลระบบ ให้กดอนุญาต จากนั้นจะขึ้นเมนูเดียวกับแบบโฟลเดอร์

  4. เลือกจากเมนู แล้วรอ

    กด Enter เฉย ๆ ได้เลย (= ตรวจทันที ไม่เก็บกราฟ) เมื่อเสร็จ รายงานจะเปิดขึ้นเองในเบราว์เซอร์

รายงานไปอยู่ที่ไหน

โฟลเดอร์รายงานจะโผล่ ข้างตัวไฟล์โดยตรง วางไฟล์ไว้ที่ไหน รายงานก็อยู่ตรงนั้น

D:\ตรวจเครื่อง\
    PMtools-1.7.2.cmd
    SRV-APP01_20260722-1305\      <- รอบนี้
        PM-Report.html
        PM-Data.json
    SRV-APP01_20260721-0930\      <- รอบก่อน แยกโฟลเดอร์ ไม่ทับกัน

ต่างจากแบบโฟลเดอร์ตรงที่ไม่มีชั้น Output\ คั่น เพราะตัวโปรแกรมทำงานอยู่ในโฟลเดอร์ชั่วคราวที่ถูกลบทิ้งเมื่อรันเสร็จ จึงต้องส่งรายงานออกมาไว้ข้างนอก

ถ้าวางไฟล์ไว้ในที่ที่เขียนไม่ได้

เช่นวางบน share แบบอ่านอย่างเดียว หรือ USB ที่ล็อกกันเขียนไว้ เครื่องมือจะไม่หยุดทำงาน แต่จะย้ายไปเขียนรายงานลงโฟลเดอร์ชั่วคราวของ Windows แทน แล้วแจ้งที่อยู่จริงไว้บนหน้าจอ ให้อ่านบรรทัด Report : ตอนจบเสมอ

แก้ชื่อหน่วยงานและเกณฑ์เตือน

ค่าตั้งต้นทั้งหมดถูกฝังอยู่ในไฟล์ ถ้าต้องการแก้ ให้ดึงออกมาก่อนหนึ่งครั้ง โดยเปิด Command Prompt ที่โฟลเดอร์นั้นแล้วสั่ง

PMtools-1.7.2.cmd -ExtractConfig

จะได้โฟลเดอร์ Config\ ออกมาวางข้างไฟล์ แก้ settings.json ด้วย Notepad ตามหัวข้อที่ 11 ได้เลย รอบถัดไปเครื่องมือจะหยิบค่าที่แก้ไว้ไปใช้เอง ไม่ต้องสั่งอะไรเพิ่ม และจะขึ้นบรรทัด Config override บอกให้ทราบว่าใช้ค่าที่แก้แล้ว

ไฟล์ไหนไม่ได้แก้ ลบทิ้งได้

ใน Config\ ที่ดึงออกมามีสามไฟล์ ถ้าแก้แค่ settings.json จะลบอีกสองไฟล์ทิ้งก็ได้ เฉพาะไฟล์ที่หายไปเครื่องมือจะใช้ของเดิมที่ฝังไว้ข้างในแทน

สั่งเฉพาะบางหัวข้อ

ใช้สวิตช์ชุดเดียวกับหัวข้อที่ 12 ได้ทั้งหมด เพียงเรียกผ่านชื่อไฟล์นี้แทน

PMtools-1.7.2.cmd -Only DISK,CERT
PMtools-1.7.2.cmd -Skip EVT
PMtools-1.7.2.cmd -MonitorMinutes 30
ยังอ่านค่าอย่างเดียวเหมือนเดิมทุกประการ

ไฟล์เดียวนี้เป็นเพียงภาชนะ ข้างในคือสคริปต์ชุดเดิมทุกบรรทัด ไม่ได้เขียนขึ้นใหม่ ตอนรันจะแตกตัวเองลงโฟลเดอร์ชั่วคราวของ Windows เรียกสคริปต์เดิม แล้วลบโฟลเดอร์นั้นทิ้งเมื่อจบเสมอ รวมถึงตอนที่มีหัวข้อใดทำงานพลาด สิ่งเดียวที่เหลือไว้บนเครื่องคือโฟลเดอร์รายงาน

สำหรับผู้ดูแลเครื่องมือ

ไฟล์นี้สร้างจากโฟลเดอร์ต้นฉบับด้วยคำสั่งเดียว รายละเอียดอยู่ใน DISTRIBUTION.md หัวข้อ แจกแบบไฟล์เดียว

.\Build-PMSingle.ps1

ห้ามแก้ไฟล์ที่ได้ด้วยมือ ถ้าต้องแก้อะไรให้แก้ที่โฟลเดอร์ต้นฉบับแล้วสั่งสร้างใหม่ เพราะขั้นตอนสร้างไม่ได้แปลงโค้ดใด ๆ เพิ่มหัวข้อตรวจใหม่แล้วสั่งสร้างซ้ำได้ทันที

5ส่งออกเป็น Word (.docx)

PM-Report.html เปิดดูและสั่งพิมพ์เป็น PDF ได้อยู่แล้ว แต่บางหน่วยงานต้องการไฟล์ Word เพื่อแก้ไขเพิ่มเติมก่อนแนบหนังสือราชการ Export-PMDocxReport.ps1 แปลง PM-Data.json ที่ตรวจไว้แล้ว ให้เป็นเอกสาร Word สองภาษาไทย/อังกฤษ หน้าตาเทียบเท่ารายงาน HTML

เครื่องมือนี้แยกจากตัวตรวจ และรันคนละเครื่องได้

ตัวตรวจ (Start-PMCheck.ps1 หรือไฟล์เดียวจากหัวข้อที่ 4) รันบนเครื่องแม่ข่ายที่ต้องการตรวจ อ่านค่าอย่างเดียวไม่ต้องพึ่งโปรแกรมอะไรเพิ่ม ส่วนตัวแปลง Word รันบนเครื่องของผู้ดูแลระบบเอง (เครื่องที่มี Microsoft Word ติดตั้งอยู่) สิ่งที่ต้องย้ายข้ามเครื่องมีแค่ไฟล์ PM-Data.json ไฟล์เดียว คัดลอกผ่านอีเมล แชร์โฟลเดอร์ หรือ USB ก็ได้

วิธีใช้

  1. คัดลอก PM-Data.json มาที่เครื่องของตัวเอง

    ไฟล์นี้อยู่ในโฟลเดอร์รายงานที่ได้จากการตรวจ ข้าง ๆ PM-Report.html เสมอ

  2. เปิด PowerShell ที่โฟลเดอร์ PMtools แล้วสั่ง
    .\Export-PMDocxReport.ps1 -DataPath D:\ผลตรวจ\PM-Data.json -OpenReport
  3. ได้ไฟล์ PM-Report.docx

    สร้างไว้ข้าง ๆ ไฟล์ PM-Data.json ที่อ่านโดยอัตโนมัติ หรือระบุปลายทางเองด้วย -OutputPath

    .\Export-PMDocxReport.ps1 -DataPath D:\ผลตรวจ\PM-Data.json -OutputPath D:\ส่งราชการ\ฉบับสมบูรณ์.docx
ต้องมี Microsoft Word ติดตั้งอยู่บนเครื่องที่รันคำสั่งนี้

เครื่องมือสั่งงาน Word ผ่าน COM automation เพื่อสร้างเอกสาร ถ้าไม่มี Word ติดตั้งอยู่ จะขึ้นข้อความแจ้งทันทีตั้งแต่เริ่มรัน ไม่ต้องใช้สิทธิ์ผู้ดูแลระบบ และไม่ขอยกระดับสิทธิ์ เพราะแค่อ่านไฟล์ที่มีอยู่แล้วกับสั่งงาน Word เท่านั้น ไม่แตะค่าอะไรบนเครื่อง

ชื่อหน่วยงานมาจากไหน

ชื่อหน่วยงาน ส่วนงาน ชื่อระบบ และผู้ตรวจสอบ อ่านจาก Config\settings.json ของเครื่องที่รันตัวแปลงนี้ ไม่ใช่จากไฟล์ PM-Data.json — ไฟล์นั้นไม่ได้เก็บค่ากลุ่มนี้ไว้โดยตั้งใจ แก้ค่าที่ Config\settings.json ตามหัวข้อที่ 11 ได้ตามปกติ ก่อนรันคำสั่งข้างต้น

ข้อจำกัดของรุ่นนี้

กราฟแนวโน้ม CPU/หน่วยความจำ (จากหัวข้อ TREND ในหัวข้อที่ 7) ยังไม่วาดในเอกสาร Word ตารางข้อมูลตัวเลข (ต่ำสุด/เฉลี่ย/เปอร์เซ็นไทล์ที่ 95/สูงสุด) ยังอยู่ครบ เพียงแต่กราฟเส้นแบบโต้ตอบได้ ยังมีเฉพาะในรายงาน HTML เท่านั้น เอกสาร Word จะมีข้อความกำกับไว้ให้ทราบตรงจุดนั้น

7วิธีอ่านรายงาน

รายงานมี 3 ส่วน

ส่วนสำหรับใครบอกอะไร
สรุปผลสำหรับผู้บริหารผู้บริหาร ผู้อนุมัติ นับจำนวนหัวข้อตามระดับ พร้อมประโยคสรุปภาพรวมหนึ่งย่อหน้า อ่านแค่ส่วนนี้ก็เข้าใจสถานการณ์แล้ว
ภาพรวมรายหัวข้อผู้ตรวจสอบ หัวหน้างาน ตารางหัวข้อทั้งหมด สถานะ และสรุปย่อว่าพบอะไร คลิกที่ชื่อหัวข้อเพื่อกระโดดไปดูรายละเอียด
รายละเอียดผลการตรวจสอบเจ้าหน้าที่เทคนิค ตารางข้อมูลจริงของแต่ละหัวข้อ เช่น รายไดรฟ์ รายบริการ รายใบรับรอง

ความหมายของสถานะ

สถานะหมายความว่าควรทำอย่างไร
✗วิกฤต / Critical พบสภาพที่กระทบการให้บริการแล้วหรือกำลังจะกระทบ เร่งดำเนินการแก้ไข ไม่ควรรอรอบบำรุงรักษาถัดไป
▲ควรเฝ้าระวัง / Warning ยังไม่กระทบการให้บริการ แต่เกินเกณฑ์ที่กำหนดไว้ วางแผนแก้ไขในรอบบำรุงรักษาถัดไป
✓ปกติ / Normal อยู่ในเกณฑ์ปกติทั้งหมด ไม่ต้องดำเนินการ
●ข้อมูล / Information ข้อมูลประกอบการวิเคราะห์ ไม่มีเกณฑ์ตัดสิน ใช้ประกอบเมื่อต้องวินิจฉัยปัญหา
!ตรวจไม่สำเร็จ / Check Failed หัวข้อนั้นอ่านข้อมูลไม่ได้ มักเกิดจากสิทธิ์ไม่พอ ลองรันใหม่ด้วยสิทธิ์ผู้ดูแลระบบ อย่าตีความว่าปกติ
สถานะไม่ได้บอกด้วยสีอย่างเดียว

ทุกป้ายสถานะมีทั้งสี สัญลักษณ์ และข้อความกำกับ จึงอ่านได้ถูกต้องแม้พิมพ์ขาวดำ หรือผู้อ่านมีภาวะตาบอดสี

เรื่องวันที่

หัวรายงานแสดงวันที่เป็น พ.ศ. คู่กับ ค.ศ. ส่วนวันที่ในตารางข้อมูลใช้รูปแบบ yyyy-MM-dd ซึ่งเป็น ค.ศ. เพราะช่องตารางมีค่าเดียวแต่มีผู้อ่านสองภาษา ถ้าใส่ 2569 ลอย ๆ โดยไม่กำกับจะกำกวมสำหรับผู้อ่านอีกฝั่ง

8กราฟแนวโน้ม CPU และหน่วยความจำ

หัวข้อ PERF เป็นการอ่านค่า ณ วินาทีที่ตรวจ ซึ่งตอบไม่ได้ว่าภาระงานสูงต่อเนื่องหรือแค่กระชากชั่วครู่ ถ้าต้องการคำตอบนั้น ให้เลือกเก็บข้อมูลจากเมนูก่อน แล้วหัวข้อ TREND จะแสดงกราฟให้เอง

กราฟบอกอะไร

  • เส้นสีน้ำเงินคือหน่วยประมวลผล เส้นสีเขียวคือหน่วยความจำที่ใช้ ทั้งคู่เป็นเปอร์เซ็นต์บนแกนเดียวกัน
  • เลื่อนเมาส์ไปบนกราฟจะเห็นค่าของทุกเส้น ณ เวลานั้น
  • ใต้กราฟมีตารางสรุป ต่ำสุด / เฉลี่ย / เปอร์เซ็นไทล์ที่ 95 / สูงสุด
ทำไมตัดสินด้วยเปอร์เซ็นไทล์ที่ 95 ไม่ใช่ค่าสูงสุด

การที่ CPU กระชากถึง 100% เพียงวินาทีเดียวเป็นเรื่องปกติของเครื่องแม่ข่ายทุกเครื่อง ถ้าใช้ค่าสูงสุดตัดสินจะขึ้นเตือนแทบทุกครั้งจนไม่มีความหมาย แต่ถ้าเปอร์เซ็นไทล์ที่ 95 อยู่ที่ 90% แปลว่าภาระงานสูงจริงเกือบตลอดเวลา ซึ่งเป็นปัญหาด้านความจุที่ต้องแก้

ข้อมูลที่เก็บไว้

  • เก็บเป็นไฟล์ CSV ที่ Output\_Perf\ เปิดด้วย Excel ได้โดยตรง ถ้าอยากทำกราฟเอง
  • ตอนสร้างรายงาน ระบบจะหยิบไฟล์ล่าสุดมาใช้ ถ้าไฟล์เก่าเกิน 24 ชั่วโมงจะไม่นำมาแสดง เพราะไม่สะท้อนสภาพปัจจุบันแล้ว
  • ถ้ายังไม่เคยเก็บ หัวข้อนี้จะขึ้นเป็น ●ข้อมูล พร้อมบอกวิธีเก็บ ไม่ถือเป็นข้อผิดพลาด

ต้นทุนการเก็บ — แต่ละตัวอย่างใช้เวลาประมาณ 0.4 วินาที ที่ความถี่ 10 วินาที จึงใช้ทรัพยากรราว 4% ของหนึ่งคอร์ เป็นการเฝ้าดูภาระงานโดยแทบไม่เพิ่มภาระเอง

AVระบบป้องกันไวรัสและมัลแวร์ปิดการป้องกันแบบเรียลไทม์ หรือฐานข้อมูลไวรัสเก่าเกิน 3 วัน

CERTใบรับรองอิเล็กทรอนิกส์หมดอายุภายใน 30 วัน (วิกฤตที่ 7 วัน)
TIMEการเทียบเวลามาตรฐานเครื่องไม่ได้เทียบเวลากับแหล่งอ้างอิง
NETอุปกรณ์เครือข่ายและการตั้งค่าไม่มีการ์ดใดกำหนดเกตเวย์หรือ DNS เลย
REBOOTสถานะการรอเริ่มระบบใหม่มีรายการค้างรอการรีสตาร์ท
CONNการเชื่อมต่อเครือข่ายping ไม่ผ่าน — ปิดไว้เป็นค่าเริ่มต้น
WUรายการปรับปรุงที่ค้างติดตั้งมีรายการค้าง — ปิดไว้เป็นค่าเริ่มต้น
AGSสถานะไซต์ ArcGIS Server (เวอร์ชัน + เครื่องในไซต์ + OS/CPU/RAM แยกบรรทัดต่อเครื่อง)เครื่องในไซต์ไม่ทำงาน — ปิดไว้เป็นค่าเริ่มต้น ต้องตั้งค่าการเชื่อมต่อก่อน ดูหัวข้อที่ 10
AGSSVCบริการแผนที่: สถานะ อินสแตนซ์ จำนวนคำขอบริการที่ควรทำงานแต่หยุด — ปิดไว้เป็นค่าเริ่มต้น
AGSUSAGEรายงานการใช้งาน: คำขอสะสม เวลาตอบสนอง คำขอหมดเวลาคำขอหมดเวลา — ปิดไว้เป็นค่าเริ่มต้น
AGSDATAการเชื่อมต่อฐานข้อมูลที่ลงทะเบียนไว้กับไซต์เชื่อมต่อไม่ได้แม้แต่รายการเดียว — ปิดไว้เป็นค่าเริ่มต้น
AGSLOGบันทึกเหตุการณ์ของ ArcGIS Server ระดับ SEVERE/WARNING ย้อน 7 วันพบบันทึกระดับ SEVERE แม้แต่รายการเดียว — ปิดไว้เป็นค่าเริ่มต้น

10ตรวจ ArcGIS Server

ห้าหัวข้อ AGS, AGSSVC, AGSUSAGE, AGSDATA และ AGSLOG ตรวจสถานะ ArcGIS Server ผ่าน Administrator API ของไซต์เอง เหมาะกับหน่วยงานที่มีเครื่องแม่ข่าย GIS แยกต่างหาก เป็นการ อ่านค่าอย่างเดียวเหมือนหัวข้ออื่นทุกประการ แต่ปิดไว้เป็นค่าเริ่มต้น เพราะต้องเชื่อมต่อออกนอกเครื่องไปยัง ArcGIS Server และต้องตั้งค่าการเชื่อมต่อก่อนใช้งานครั้งแรก

  1. เปิดเมนูหลัก แล้วเลือก ArcGIS Server connection

    ดับเบิลคลิก Run-PM.cmd (หรือไฟล์เดียวจากหัวข้อที่ 4) แล้วเลือก [A] ArcGIS Server connection... จากเมนู

  2. เลือก [1] เพื่อกรอกข้อมูลการเชื่อมต่อ

    กรอกที่อยู่ไซต์ ชื่อบัญชี และรหัสผ่าน เมนูจะแสดงตัวอย่างให้ครบทั้งสามช่อง

  3. ทดสอบการเชื่อมต่อ

    เมนูมีตัวเลือกทดสอบให้ทันที จะบอกผลว่าเชื่อมต่อได้หรือไม่ พร้อมเวอร์ชันของไซต์ที่เชื่อมต่อ ถ้าเชื่อมต่อไม่ได้ ข้อความจะบอกสาเหตุตรง ๆ เช่น รหัสผ่านผิด หรือที่อยู่ไซต์ไม่ถูกต้อง

รหัสผ่านเก็บที่ %LOCALAPPDATA%\PMtools\arcgis-connection.xml เข้ารหัสด้วย Windows DPAPI ซึ่งผูกกับบัญชีผู้ใช้และเครื่องที่ตั้งค่าไว้เท่านั้น ไม่ได้เก็บใน Config\settings.json ที่ขึ้น git และแจกไปพร้อมไฟล์เดียว

ห้าหัวข้อตรวจอะไรบ้าง

รหัสตรวจอะไรขึ้นวิกฤตเมื่อไร
AGSเวอร์ชันไซต์ สถานะเครื่องทุกเครื่องในไซต์ และรายละเอียดฮาร์ดแวร์/OS แยกเป็นสามบรรทัดย่อยต่อเครื่อง (OS, CPU พร้อมจำนวนคอร์, RAM รวม/ว่าง — ข้อมูลประกอบ ไม่มีผลต่อการตัดสินวิกฤต) มีเครื่องที่ไม่ทำงาน
AGSSVCสถานะบริการแผนที่ทุกตัว จำนวนอินสแตนซ์ที่ใช้ และจำนวนคำขอสะสม บริการที่ตั้งค่าให้ทำงานแต่หยุดอยู่จริง (บริการที่ปิดไว้โดยตั้งใจไม่นับเป็นปัญหา)
AGSUSAGEคำขอสะสม เวลาตอบสนองสูงสุด และคำขอที่หมดเวลา จากรายงานที่มีอยู่แล้วบนไซต์ พร้อมกราฟจำนวนคำขอตามช่วงเวลา (ภาพรวมทั้งไซต์ ไม่แยกรายบริการ) มีคำขอที่หมดเวลาแม้แต่รายการเดียว
AGSDATAการเชื่อมต่อฐานข้อมูล/แหล่งข้อมูลที่ลงทะเบียนไว้กับไซต์ (enterprise geodatabase, file share ฯลฯ) ทดสอบทีละรายการ เชื่อมต่อไม่ได้แม้แต่รายการเดียว
AGSLOGบันทึกเหตุการณ์ระดับ SEVERE/WARNING ย้อน 7 วัน (สูงสุด 100 รายการต่อระดับ) พบบันทึกระดับ SEVERE แม้แต่รายการเดียว
AGSDATA ไม่เก็บหรือแสดงรหัสผ่านฐานข้อมูลเด็ดขาด

ไซต์จะส่งข้อมูลการเชื่อมต่อ (ชื่อเซิร์ฟเวอร์ ชื่อฐานข้อมูล ชื่อบัญชี และรหัสผ่านที่เข้ารหัสไว้) มาให้เครื่องมือใช้ทดสอบการเชื่อมต่อเท่านั้น ข้อมูลชุดนี้ไม่เคยถูกบันทึกลง PM-Data.json หรือแสดงในรายงานเลย สิ่งที่บันทึกไว้มีแค่ชื่อรายการ ประเภท และผลว่าเชื่อมต่อได้หรือไม่ได้เท่านั้น

AGSLOG ดึงข้อมูลจำกัดจำนวนเพื่อความเร็ว ไม่ใช่ตัวเลขที่ครบทุกรายการเสมอไป

ไซต์ใหญ่มีบันทึกเหตุการณ์เป็นหมื่นรายการต่อ 7 วัน หัวข้อนี้จึงดึงมาสูงสุด 100 รายการต่อระดับ ความรุนแรง (ปรับได้ที่ ArcGIS.LogPageSize) ถ้ามีมากกว่านั้นจะมีข้อความ บอกไว้ในรายงานว่าตัวเลขที่แสดงเป็นค่าอย่างน้อยที่สุด ไม่ใช่จำนวนที่ครบทั้งหมด

สั่งรัน

ทางที่ง่ายที่สุดคือใช้เมนู (หัวข้อที่ 6) แล้วเลือก [2] ArcGIS Server ตั้งแต่หน้าแรก ส่วนสั่งตรงจาก PowerShell ทำได้ดังนี้

.\Start-PMCheck.ps1 -Group ArcGIS                 # ครบทั้งห้าหัวข้อ ArcGIS ในคำสั่งเดียว
.\Start-PMCheck.ps1 -Group Server                 # ทุกหัวข้อยกเว้น ArcGIS
.\Start-PMCheck.ps1                               # ปกติ - ยังไม่รวมหัวข้อ ArcGIS จนกว่าจะเปิดไว้
Server กับ ArcGIS แยกกันเด็ดขาด ไม่รันรวมกันในรอบเดียว

เครื่องแม่ข่ายทั่วไปกับไซต์ ArcGIS Server ไม่มีเหตุผลต้องตรวจรวมกัน ถ้าสั่ง -Only ผสมหัวข้อ Server กับ ArcGIS เข้าด้วยกัน (เช่น -Only DISK,AGS) เครื่องมือจะปฏิเสธพร้อมข้อความบอกให้สั่งแยกสองรอบแทน

ถ้าต้องการให้ ArcGIS รันรวมกับหัวข้ออื่นทุกครั้งโดยไม่ต้องพิมพ์ -Group ให้ลบ "AGS", "AGSSVC", "AGSUSAGE", "AGSDATA" และ "AGSLOG" ออกจาก Checks.Disabled ใน Config\settings.json ตามหัวข้อที่ 11

ยังอ่านค่าอย่างเดียวเหมือนหัวข้ออื่นทุกประการ

ทุกหัวข้อใช้ GET เป็นหลัก ยกเว้นสี่จุดที่ API บังคับให้เป็น POST (ไม่ใช่เพราะเขียนอะไร): ขั้นตอนขอ token เพราะรหัสผ่านต้อง เดินทางใน request body และ endpoint ของ AGSDATA/AGSLOG เพราะต้องส่ง JSON ยาวเกินจะใส่ใน query string หัวข้อ AGSUSAGE อ่านได้เฉพาะรายงานที่ มีอยู่แล้วบนไซต์ จะไม่สั่งสร้างรายงานใหม่เด็ดขาด หัวข้อ AGSDATA แค่ทดสอบการเชื่อมต่อด้วย credential ที่ลงทะเบียนไว้แล้ว ไม่เคยส่งหรือรับรหัสผ่านฐานข้อมูลเอง และหัวข้อ AGSLOG แค่อ่านบันทึกที่ไซต์ เก็บไว้แล้ว ไม่ได้สั่งให้บันทึกเพิ่มหรือลบบันทึกเก่าทิ้ง

รายละเอียดเชิงลึกกว่านี้ (schema ของ API, เกณฑ์ configured เทียบ realTime ฯลฯ) อยู่ใน CHECKS.md หัวข้อ ArcGIS Server สำหรับผู้ดูแลเครื่องมือ

11การตั้งค่า

ไฟล์ตั้งค่าอยู่ในโฟลเดอร์ Config\ ทั้งหมด แก้ด้วย Notepad ได้เลย แก้แล้วมีผลทันทีในการรันครั้งถัดไป ไม่ต้องแก้สคริปต์

settings.json — เกณฑ์เตือนและข้อมูลหน่วยงาน

เกณฑ์เตือนทุกค่าอยู่ในไฟล์นี้ที่เดียว ไม่มีเกณฑ์ไหนถูกเขียนตายไว้ในสคริปต์

"DiskFreePercent": { "Warn": 20, "Crit": 10, "Direction": "Lower" }

Direction บอกว่าค่ามากหรือน้อยแบบไหนคือแย่

  • Lower — ยิ่งน้อยยิ่งแย่ เช่น พื้นที่ว่าง จำนวนวันคงเหลือก่อนใบรับรองหมดอายุ
  • Higher — ยิ่งมากยิ่งแย่ เช่น การใช้งาน CPU จำนวนวันที่ไม่ได้ติดตั้งรายการปรับปรุง

ค่าอื่นที่ปรับได้บ่อย

ค่าทำอะไร
Report.ShowRecommendationsแสดงหัวข้อข้อเสนอแนะในรายงานหรือไม่ ค่าเริ่มต้นคือ false เพื่อให้รายงานกระชับ ตั้งเป็น true เพื่อเปิดกลับ
Lookback.EventLogDaysย้อนดูบันทึกเหตุการณ์กี่วัน ค่าเริ่มต้น 7
Monitor.DefaultMinutesระยะเวลาเก็บข้อมูลของตัวเลือกที่ 3 ในเมนู ค่าเริ่มต้น 30
Checks.Disabledรายการรหัสหัวข้อที่ไม่ต้องการให้รันอัตโนมัติ ค่าเริ่มต้นคือ ["WU", "CONN"]
Backup.TaskKeywordsคำค้นชื่องานสำรองข้อมูล ถ้าใช้ระบบสำรองยี่ห้ออื่นให้เพิ่มคำที่นี่
ข้อเสนอแนะไม่ได้หายไปไหน

แม้ปิดการแสดงผลในรายงาน ข้อเสนอแนะฉบับเต็มยังถูกคำนวณและเก็บไว้ใน PM-Data.json ครบทุกครั้ง จึงย้อนไปดูได้ตลอดโดยไม่ต้องรันใหม่

services.json — บริการสำคัญที่ต้องเฝ้าระวัง

ใส่ชื่อบริการสำคัญของระบบนั้น ๆ ลงใน Watchlist ถ้าบริการในรายการหยุดทำงานจะถูกจัดเป็นระดับวิกฤต แทนที่จะเป็นแค่ควรเฝ้าระวัง

"Watchlist": ["W3SVC", "MSSQLSERVER", "SQLSERVERAGENT"]

ถ้าปล่อยว่างไว้ก็ใช้งานได้ทันที เพราะเครื่องมือจะตรวจหาบริการที่ตั้งค่าเริ่มอัตโนมัติแต่หยุดอยู่ให้เองอยู่แล้ว ในไฟล์มีตัวอย่างชื่อบริการที่พบบ่อยแยกตามบทบาทของเครื่องให้คัดลอกไปใช้

ต้องใส่ชื่อบริการ ไม่ใช่ชื่อที่แสดง

ดูชื่อจริงได้ด้วยคำสั่ง Get-Service | Select-Object Name, DisplayName, Status โดยคอลัมน์ Name คือชื่อที่ต้องใส่

i18n.json — ข้อความทุกคำในรายงาน

ข้อความทุกคำที่ปรากฏในรายงานอยู่ในไฟล์นี้ ทั้งภาษาไทยและอังกฤษ ถ้าต้องการเปลี่ยนสำนวนให้ตรงกับที่หน่วยงานใช้ แก้ที่นี่ได้เลย

ห้ามลบหรือสลับลำดับเครื่องหมาย {0} {1}

เครื่องหมายเหล่านี้คือช่องเติมค่าที่ระบบจะแทนตัวเลขหรือชื่อลงไป ถ้าลบออกหรือสลับลำดับ ข้อความจะแสดงผิดหรือขึ้นเป็นข้อความดิบ

12สั่งงานจาก PowerShell

สำหรับผู้ที่ต้องการควบคุมเพิ่มเติม หรือต้องการนำไปตั้งเป็นงานตามกำหนดเวลา สามารถสั่งตรงได้โดยไม่ผ่านเมนู

.\Start-PMCheck.ps1                     # ตรวจตามปกติ
.\Start-PMCheck.ps1 -OpenReport         # ตรวจแล้วเปิดรายงานให้เลย
.\Start-PMCheck.ps1 -Only DISK,CERT     # ตรวจเฉพาะบางหัวข้อ
.\Start-PMCheck.ps1 -Skip NET           # ตรวจทุกหัวข้อยกเว้นที่ระบุ
.\Start-PMCheck.ps1 -MonitorMinutes 30  # เก็บข้อมูล 30 นาทีแล้วตรวจ
.\Start-PMCheck.ps1 -Only WU            # ตรวจรายการปรับปรุงที่ค้าง (ช้า)

.\Start-PMMonitor.ps1 -Minutes 30       # เก็บข้อมูลอย่างเดียว
.\Start-PMMonitor.ps1 -Minutes 5 -IntervalSeconds 2   # ปรับความถี่

รหัสสถานะเมื่อรันจบ

ใช้ต่อยอดกับ Scheduled Task หรือระบบเฝ้าระวังได้

รหัสหมายความว่า
0ปกติทั้งหมด ไม่มีสิ่งที่ต้องดำเนินการ
1พบประเด็นที่ควรเฝ้าระวัง
2พบประเด็นวิกฤต หรือมีหัวข้อที่ตรวจไม่สำเร็จ

13ปัญหาที่พบบ่อย

ดับเบิลคลิกแล้วหน้าต่างปิดทันที

มักเกิดจากนโยบายการรันสคริปต์ ให้ลองเปิด PowerShell ด้วยสิทธิ์ผู้ดูแลระบบแล้วสั่ง

powershell -NoProfile -ExecutionPolicy Bypass -File "D:\PMtools\Show-PMMenu.ps1"

ข้อความผิดพลาดที่ขึ้นจะบอกสาเหตุที่แท้จริง

ได้ไฟล์เดียวมา ดับเบิลคลิกแล้วขึ้นหน้าต่างสีฟ้าว่า Windows protected your PC

เป็นเพราะไฟล์ถูกติดเครื่องหมายว่ามาจากเครื่องอื่น ให้กด More info แล้ว Run anyway หรือจัดการให้จบครั้งเดียวด้วยการคลิกขวาที่ไฟล์ → Properties → ติ๊ก Unblock → OK ก่อนดับเบิลคลิก ดูหัวข้อที่ 4

ได้ไฟล์เดียวมา รันแล้วหาโฟลเดอร์รายงานไม่เจอ

ตามปกติโฟลเดอร์รายงานจะอยู่ข้างตัวไฟล์ ถ้าไม่เจอ แปลว่าที่วางไฟล์นั้นเขียนไม่ได้ เครื่องมือจึงย้ายไปเขียนที่โฟลเดอร์ชั่วคราวแทน ให้อ่านบรรทัด Report : บนหน้าจอตอนจบ ซึ่งบอกที่อยู่จริงไว้ แล้วย้ายไฟล์วางที่เขียนได้ก่อนรันรอบต่อไป

ภาษาไทยในรายงานเป็นตัวประหลาด

เกิดจากเบราว์เซอร์เดาการเข้ารหัสผิด ซึ่งไม่ควรเกิดเพราะไฟล์รายงานฝังข้อมูลการเข้ารหัสไว้แล้ว ถ้าเจอ ให้ลองเปิดด้วยเบราว์เซอร์อื่น และตรวจว่าไฟล์ถูกคัดลอกมาครบไม่เสียหายระหว่างทาง

รายงานขึ้นแถบเตือนสีเหลืองว่าไม่มีสิทธิ์ผู้ดูแลระบบ

แปลว่ารันโดยไม่ได้ยกระดับสิทธิ์ ข้อมูลบางหัวข้อจะไม่ครบ ให้ปิดแล้วเปิดใหม่ผ่าน Run-PM.cmd ซึ่งจะขอสิทธิ์ให้เอง

ในการรันครั้งแรกเป็นเรื่องปกติ เพราะระบบรายงานทุกบริการที่ตั้งค่าให้เริ่มอัตโนมัติแต่หยุดอยู่ ซึ่งบางตัวถูกปิดโดยตั้งใจ วิธีจัดการคือ ใส่เฉพาะบริการที่สำคัญจริงลงใน Config\services.json บริการเหล่านั้นจะยกระดับเป็นวิกฤต ส่วนที่เหลือยังเป็นแค่ควรเฝ้าระวัง ให้ตรวจดูตามสมควร

หัวข้อการสำรองข้อมูลบอกว่าไม่พบร่องรอย

เครื่องมือตรวจจากบันทึกของ Windows Server Backup และงานตามกำหนดเวลาที่ชื่อเข้าเกณฑ์ ถ้าหน่วยงานใช้ระบบสำรองข้อมูลยี่ห้ออื่น ให้เพิ่มคำค้นใน Backup.TaskKeywords หรือหากการสำรองข้อมูลทำจากระบบภายนอกเครื่องนี้ ก็ถือว่าเป็นผลปกติ

เปิดหัวข้อ ArcGIS แล้วขึ้นว่า "No ArcGIS Server connection is configured"

แปลว่ายังไม่เคยตั้งค่าการเชื่อมต่อบนเครื่องนี้ ให้เปิดเมนูหลักแล้วเลือก [A] ArcGIS Server connection... ก่อน ดูหัวข้อที่ 10 การตั้งค่าผูกกับบัญชี Windows และเครื่องที่ตั้งไว้เท่านั้น ถ้าเปลี่ยนไปรันด้วยบัญชีอื่นหรือย้ายไปอีกเครื่อง ต้องตั้งค่าใหม่

ระหว่างรันแล้วเครือข่ายไม่เสถียร

เคยพบอาการนี้บนเครื่องที่ทำ NIC Teaming และได้แก้ไขแล้ว โดยเปลี่ยนไปอ่านข้อมูลเครือข่าย ด้วยวิธีที่ไม่แตะไดรเวอร์การ์ดเครือข่าย หากยังพบอาการอยู่ ให้ลองตัดหัวข้อเครือข่ายออกเพื่อยืนยัน

ยังไม่ได้ทดสอบบน Windows Server จริง

เครื่องมือนี้พัฒนาและทดสอบบน Windows 11 กับ PowerShell 5.1 ซึ่งเป็น PowerShell รุ่นเดียวกับที่ใช้บน Windows Server แต่ยังไม่เคยรันบนเครื่องแม่ข่ายจริง จึงควรทดลองบนเครื่องทดสอบก่อนนำไปใช้กับระบบสำคัญ

จุดที่ควรตรวจดูเป็นพิเศษเมื่อนำไปวางบนเครื่องแม่ข่ายครั้งแรก

หัวข้อสิ่งที่อาจต่างไปเครื่องมือรับมืออย่างไร
AV แหล่งข้อมูลที่ใช้บน Windows รุ่นทั่วไปไม่มีบน Windows Server มีทางสำรองสามชั้น หากตรวจไม่พบจริง จะรายงานตรง ๆ ว่าตรวจไม่พบ ไม่เดาว่าปลอดภัย
BACKUP บันทึกของ Windows Server Backup มีเฉพาะเมื่อติดตั้งส่วนประกอบนั้นแล้ว รายงานว่า "ไม่พบร่องรอย" ไม่ใช่ "ไม่มีการสำรองข้อมูล" เพราะอาจสำรองจากระบบภายนอก
TIME คำสั่งเทียบเวลาแสดงข้อความตามภาษาของเครื่อง อ่านจากลำดับบรรทัดและตัวเลขแทนข้อความ ถ้าอ่านไม่ได้จะรายงานเป็นข้อมูล ไม่เดาว่าผิดปกติ

สิ่งที่เครื่องมือนี้ยังไม่ครอบคลุม

  • การตรวจตามบทบาทเฉพาะทาง เช่น Active Directory, DNS, DHCP, IIS, SQL Server, Hyper-V
  • การตรวจสอบด้านความมั่นคงปลอดภัยและการปฏิบัติตามมาตรฐาน
  • สถานะอุปกรณ์ฮาร์ดแวร์ เช่น RAID แหล่งจ่ายไฟ อุณหภูมิ
  • การเปรียบเทียบผลย้อนหลังข้ามรอบการตรวจ
  • การเก็บข้อมูลจากหลายเครื่องแบบรวมศูนย์

โครงสร้างของเครื่องมือออกแบบให้เพิ่มหัวข้อใหม่ได้โดยไม่ต้องแก้ไฟล์เดิม รายละเอียดสำหรับผู้พัฒนาอยู่ในไฟล์ CHECKS.md และ HANDOVER.md
