# PMtools — เครื่องมือบำรุงรักษาเชิงป้องกัน Windows Server

เก็บสถานะเครื่องแม่ข่ายทุกหัวข้อที่ควรรู้ในรอบเดียว แล้วออกไฟล์รายงาน HTML สองภาษา (ไทย/อังกฤษ)
ที่ส่งให้ผู้บริหารหรือหน่วยงานราชการได้โดยตรง หรือสั่ง Print to PDF แนบหนังสือได้ทันที

**[ดาวน์โหลดรุ่นล่าสุด](https://github.com/tongkhemmathat-dot/gispmtools/releases/latest)**
— ไฟล์เดียวจบ (`.cmd`/`.zip`) พร้อมใช้งาน ไม่ต้อง clone ทั้ง repo ดูวิธีใช้ที่
[docs/DISTRIBUTION.md](docs/DISTRIBUTION.md)

## อ่านค่าอย่างเดียว ไม่แก้ไขระบบ

เครื่องมือนี้ **ไม่เปลี่ยนค่าใด ๆ บนเครื่องแม่ข่าย** ทุกหัวข้อเป็นการอ่านสถานะเท่านั้น
ไม่ตั้งค่า ไม่แก้ registry ไม่เริ่ม/หยุดบริการ ไม่ติดตั้งอะไร และ**ไม่รีสตาร์ทเครื่อง**
สิ่งเดียวที่เขียนลงดิสก์คือโฟลเดอร์รายงานใน `Output\`

มีเจ็ดหัวข้อที่ติดต่อออกนอกเครื่อง (`WU` ถาม WSUS/Microsoft Update, `CONN` ส่ง ping,
`AGS`/`AGSSVC`/`AGSUSAGE`/`AGSDATA`/`AGSLOG` ตรวจ **ArcGIS Server** ผ่าน Administrator API แบบอ่านอย่างเดียว)
ทั้งเจ็ด**ปิดไว้เป็นค่าเริ่มต้น** เปิดใช้เฉพาะครั้งได้ด้วย `-Only WU` หรือ `-Group ArcGIS`
เป็นต้น หัวข้อ ArcGIS ต้องตั้งค่าการเชื่อมต่อก่อนใช้งานครั้งแรกด้วย (เมนูหลัก ->
`ArcGIS Server connection...`) และไม่รันรวมกับหัวข้อ Server ในรอบเดียวกัน
รายละเอียดครบอยู่ใน [docs/CHECKS.md](docs/CHECKS.md#arcgis-server)

## เริ่มใช้งาน

### วิธีที่เร็วที่สุด — ไฟล์เดียว ไม่ต้อง clone (แนะนำ)

1. [ดาวน์โหลด `.cmd`](https://github.com/tongkhemmathat-dot/gispmtools/releases/latest) ไปวางบนเครื่องแม่ข่าย — ไฟล์เดียว ไม่ต้องเอาอะไรไปด้วย
2. ดับเบิลคลิกไฟล์นั้นตรง ๆ แล้วกดอนุญาตเมื่อระบบขอสิทธิ์ผู้ดูแลระบบ ขึ้นเมนูให้เลือกทันที

หรือสั่งจาก command line รอบเดียวจบ ไม่ผ่านเมนู (เหมาะกับ Scheduled Task หรือรันจากระยะไกล):

```cmd
PMtools-1.7.3.cmd -Only DISK,CERT -OpenReport
```

ผลลัพธ์ (`<ชื่อเครื่อง>_<วันเวลา>\PM-Report.html`) จะออกมาข้างไฟล์ `.cmd` นั้นโดยตรง
ไม่ต้องมีไฟล์อื่นประกอบเลย รายละเอียดสวิตช์อื่น ๆ (`-ExtractConfig` แก้ค่าตั้งต้น ฯลฯ)
อยู่ใน **[docs/DISTRIBUTION.md](docs/DISTRIBUTION.md)**

### วิธีแบบทั้งโฟลเดอร์ — เหมาะกับผู้ที่อยากแก้โค้ดหรือดูซอร์ส

1. คัดลอกโฟลเดอร์ `PMtools` ทั้งโฟลเดอร์ไปวางบนเครื่องแม่ข่าย
2. เปิด `Config\settings.json` แก้ชื่อหน่วยงานและชื่อผู้ตรวจสอบ (ทำครั้งเดียว)
3. ดับเบิลคลิก **`Run-PM.cmd`** แล้วกดอนุญาตเมื่อระบบขอสิทธิ์ผู้ดูแลระบบ
4. เลือกจากเมนูว่าจะเก็บกราฟ CPU/หน่วยความจำก่อนกี่นาที (กด Enter เฉย ๆ = ตรวจทันที)
5. รายงานจะเปิดขึ้นเองในเบราว์เซอร์ กด `Ctrl + P` เลือก **Save as PDF** เพื่อได้ไฟล์แนบหนังสือ

หรือสั่งตรงจาก PowerShell:

```powershell
.\Start-PMCheck.ps1 -OpenReport
```

ขั้นตอนละเอียด รวมถึงหน้าตาเมนูและสวิตช์อื่น ๆ อยู่ใน **[docs/USAGE.md](docs/USAGE.md)**

ผลลัพธ์อยู่ที่ `Output\<ชื่อเครื่อง>_<วันเวลา>\PM-Report.html` (คู่กับ `PM-Data.json` ที่เก็บข้อมูลดิบไว้ครบ)
