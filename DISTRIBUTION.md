# แจกจ่าย PMtools

ภาพรวมย่อและลิงก์ไปเอกสารอื่นอยู่ใน [README.md](README.md) — ไฟล์นี้ครอบคลุมสองวิธีส่งมอบผลลัพธ์
ที่ไม่ใช่การรันแล้วเปิด `PM-Report.html` แบบปกติ: การแจกตัวเครื่องมือเป็นไฟล์เดียว
และการแปลงผลตรวจที่ได้ไปเป็นเอกสาร Word

---

## แจกแบบไฟล์เดียว

การคัดลอกทั้งโฟลเดอร์ 24 ไฟล์ไปเครื่องแม่ข่ายมักเป็นขั้นตอนที่เสียเวลาที่สุด
โดยเฉพาะเวลาส่งทางอีเมลหรือผ่านเครื่องที่เข้าถึงได้จำกัด
`Build-PMSingle.ps1` จะรวมทุกอย่างเป็น **ไฟล์เดียว** ที่แตกตัวเองได้

```powershell
.\Build-PMSingle.ps1              # ได้ Dist\PMtools-1.7.0.cmd (~95 KB)
.\Build-PMSingle.ps1 -Format Both # ได้ทั้ง .cmd และ .ps1
```

จากนั้นคัดลอกไฟล์นั้นไฟล์เดียวไปเครื่องแม่ข่าย **ไม่ต้องเอาอะไรไปด้วย**

| ทำอะไร | สั่งอย่างไร |
|---|---|
| ตรวจตามปกติ | ดับเบิลคลิก `PMtools-1.7.0.cmd` (ขอสิทธิ์ผู้ดูแลระบบให้เอง แล้วขึ้นเมนูเดิม) |
| ตรวจเฉพาะหัวข้อ | `PMtools-1.7.0.cmd -Only DISK,CERT` |
| แก้ค่าตั้งต้น | `PMtools-1.7.0.cmd -ExtractConfig` แล้วแก้ `Config\settings.json` ที่ได้มา |

โฟลเดอร์รายงาน `<ชื่อเครื่อง>_<วันเวลา>\` ออก**ข้างตัวไฟล์โดยตรง** ไม่มีชั้น `Output\` คั่นเหมือนแบบโฟลเดอร์
เพราะตัวโปรแกรมอยู่ในโฟลเดอร์ชั่วคราวที่ถูกลบทิ้งหลังรันเสร็จ วางไฟล์ไว้ที่ไหน รายงานก็โผล่ตรงนั้น

```
D:\ตรวจเครื่อง\
    PMtools-1.7.0.cmd
    SRV-APP01_20260722-1305\      <- รอบนี้
        PM-Report.html
        PM-Data.json
    SRV-APP01_20260721-0930\      <- รอบก่อน แยกโฟลเดอร์ ไม่ทับกัน
```

ถ้าที่วางไฟล์เขียนไม่ได้ เช่นวางบน share แบบอ่านอย่างเดียวหรือ USB ที่ล็อกไว้
จะย้ายไปเขียนที่โฟลเดอร์ชั่วคราวแทนและแจ้งให้ทราบ — ตรวจด้วยการลองเขียนไฟล์จริง
ไม่ใช่แค่ดูว่าโฟลเดอร์มีอยู่ไหม เพราะต้องรู้ตั้งแต่ก่อนตรวจ ไม่ใช่ตอนจะเขียนรายงาน

**ยังอ่านค่าอย่างเดียวเหมือนเดิม** ไฟล์นี้เป็นแค่ภาชนะ ข้างในคือสคริปต์ชุดเดิมทุกบรรทัด
ตอนรันมันจะแตกตัวเองลงโฟลเดอร์ชั่วคราว เรียกสคริปต์เดิม แล้วลบโฟลเดอร์นั้นทิ้งเมื่อจบ
(รวมถึงตอนที่มีหัวข้อใดพัง) สิ่งเดียวที่เหลือไว้คือโฟลเดอร์รายงาน

### เรื่องที่ควรรู้

- **`.cmd` แก้ปัญหา execution policy ให้เอง** — มันเรียก `powershell -ExecutionPolicy Bypass`
  ให้เฉพาะ process ตัวเอง ไม่ได้แก้ค่าอะไรบนเครื่อง และ `.cmd` คลิกขวาสั่ง
  *Run as administrator* ได้ ซึ่ง `.ps1` ทำไม่ได้
- **`.ps1` มีไว้สำหรับ Task Scheduler หรือ pipeline** ที่เรียก `powershell.exe` เองอยู่แล้ว
  และไม่ต้องการหน้าต่างขอสิทธิ์ — สั่งด้วย `powershell -ExecutionPolicy Bypass -File PMtools-1.7.0.ps1`
- **ถ้าส่งไฟล์ผ่านอีเมลหรือดาวน์โหลดมา** Windows จะติดเครื่องหมายว่ามาจากเครื่องอื่น
  ให้คลิกขวาที่ไฟล์ → Properties → ติ๊ก **Unblock** ก่อนรัน
  (ไฟล์ในรูปแบบ `.zip` มักผ่านตัวกรองอีเมล/เบราว์เซอร์ได้ง่ายกว่า `.cmd` เปล่า ๆ —
  ดาวน์โหลดหน้า [Releases ของ repo นี้](https://github.com/tongkhemmathat-dot/gispmtools/releases)
  แล้วเลือกไฟล์ `.zip` ได้เลย)
- **ไฟล์ที่ได้ห้ามแก้มือ** ให้แก้ที่ซอร์สแล้ว build ใหม่ — ไม่มีขั้นตอนแปลงโค้ดใด ๆ
  ระหว่าง build ดังนั้นเพิ่มหัวข้อตรวจใหม่แล้ว build ซ้ำได้เลย ไม่ต้องแตะ `Build-PMSingle.ps1`
- ข้อความภาษาไทยใน `Config\*.json` ปลอดภัย เพราะ payload เก็บเป็น base64 ซึ่งตรงทุกไบต์

## ส่งออกเป็น Word (.docx)

`PM-Report.html` เปิดดูและสั่งพิมพ์ PDF ได้อยู่แล้ว แต่บางหน่วยงานต้องการไฟล์ Word
เพื่อแก้ไขเพิ่มเติมก่อนแนบหนังสือราชการ — `Export-PMDocxReport.ps1` แปลง `PM-Data.json`
ที่ตรวจไว้แล้วให้เป็นเอกสาร Word สองภาษาไทย/อังกฤษ หน้าตาเทียบเท่ารายงาน HTML

**เครื่องมือนี้แยกจากตัวตรวจ และรันคนละเครื่องได้** ตัวตรวจ (`Start-PMCheck.ps1` หรือไฟล์เดียว)
รันบนเครื่องแม่ข่ายที่ต้องการตรวจ อ่านค่าอย่างเดียวไม่ต้องพึ่งโปรแกรมอะไรเพิ่ม
ส่วนตัวแปลง Word รันบนเครื่องของผู้ดูแลระบบเอง (เครื่องที่มี Microsoft Word ติดตั้งอยู่)
สิ่งที่ต้องย้ายข้ามเครื่องมีแค่ไฟล์ `PM-Data.json` ไฟล์เดียว (คัดลอกผ่านอีเมล แชร์โฟลเดอร์ หรือ USB)

```powershell
.\Export-PMDocxReport.ps1 -DataPath D:\ผลตรวจ\PM-Data.json
.\Export-PMDocxReport.ps1 -DataPath D:\ผลตรวจ\PM-Data.json -OpenReport
.\Export-PMDocxReport.ps1 -DataPath D:\ผลตรวจ\PM-Data.json -OutputPath D:\ส่งราชการ\ฉบับสมบูรณ์.docx
```

ค่าเริ่มต้นจะสร้าง `PM-Report.docx` ไว้ข้าง ๆ ไฟล์ `PM-Data.json` ที่อ่าน

**เอกสารที่ได้เป็นไปตามมาตรฐานการพิมพ์หนังสือราชการไทย** — ฟอนต์ TH Sarabun New ขนาด 16pt
สำหรับเนื้อหา (ตารางลดลงมาที่ 14pt เพื่อให้พอดีหน้ากระดาษ) ขอบกระดาษซ้าย 3 ซม. ขวา 2 ซม.
บน-ล่าง 2.5 ซม. ตามระเบียบสำนักนายกรัฐมนตรี ว่าด้วยงานสารบรรณ พ.ศ. 2526 และคู่มือการพิมพ์
หนังสือราชการที่ออกตามระเบียบนั้น — ต้องมี**ฟอนต์ TH Sarabun New ติดตั้งอยู่บนเครื่องที่รัน**
(มักติดมากับเครื่องราชการ/หน่วยงานไทยอยู่แล้ว ดาวน์โหลดเพิ่มได้ฟรีถ้ายังไม่มี)

**ต้องการ Microsoft Word ติดตั้งอยู่บนเครื่องที่รันเท่านั้น** ไม่ต้องใช้สิทธิ์ผู้ดูแลระบบ
และไม่ขอยกระดับสิทธิ์ เพราะแค่อ่านไฟล์ที่มีอยู่แล้วกับสั่งงาน Word ผ่าน COM automation
ไม่แตะค่าอะไรบนเครื่อง

ชื่อหน่วยงาน / ส่วนงาน / ชื่อระบบ / ผู้ตรวจสอบ **อ่านจาก `Config\settings.json` ของเครื่องที่รันตัวแปลงนี้**
ไม่ใช่จากไฟล์ `PM-Data.json` (ไฟล์นั้นไม่ได้เก็บค่าพวกนี้ไว้โดยตั้งใจ) — แก้ค่าที่ `Config\settings.json`
ตามหัวข้อ *การตั้งค่า* ใน [CHECKS.md](CHECKS.md) ได้ตามปกติ ก่อนรันคำสั่งข้างบน

**ข้อจำกัดของ v1**: กราฟแนวโน้ม CPU/หน่วยความจำ (จากหัวข้อ TREND) ยังไม่วาดในเอกสาร Word
ตารางข้อมูลตัวเลข (ต่ำสุด/เฉลี่ย/เปอร์เซ็นไทล์ที่ 95/สูงสุด) ยังอยู่ครบ เพียงแต่กราฟเส้นแบบโต้ตอบได้
ยังมีเฉพาะในรายงาน HTML เท่านั้น เอกสาร Word จะมีข้อความกำกับไว้ให้ทราบตรงจุดนั้น
